Chapter 1 Linux Commands

What behind the # is the explanation of the command.
demo.txt or demo.zip is a file name.
demo or desktop is a directory name.

1.1 Basic Commands

pwd

pwd # check the path from the root to the current directory

cd

cd / # change directory to the root directory
cd .. # change directory to the upper directory

ls

ls # simply check the names of all the files or directories inside of current directory
ls -l # more detailed information will be presented with the names of all the files or directories inside of current directory
ls -la # more detailed information will be presented with the names of all the files or directories (including hidden files or directories) inside of current directory

Hidden files or directories start with “.”, such as “.demo.txt” or “.demo”


touch

touch demo.txt # create a file named demo.txt

mv

mv demo.txt ~/desktop # move the demo.txt from the current directory to the desktop
mv demo.txt demo1.txt # rename the demo.txt file to demo1.txt
mv demo.txt ~/desktop/demo1.txt # move the demo.txt from the current directory to the desktop and rename the demo.txt file to demo1.txt

cat

cat demo.txt # check the content of demo.txt

head

head demo.txt # check the 10 lines from the beginning of demo.txt
head -n 20 demo.txt # check the 20 lines from the beginning of demo.txt

tail

tail demo.txt # check the 10 lines from the end of demo.txt
tail -n 20 demo.txt # check the 20 lines from the end of demo.txt
tail -f demo.txt # check automatically the tail of the demo.txt if there are new contents added to the demo.txt

One example of using “tail -f” is when you want to check a log file.


mkdir

mkdir demo # make a directory called demo
mkdir -p demo1/demo2/demo3/demo4 # make a directory called demo1 that contains demo2 that contains demo3 that contains demo4

rm

rm demo.txt # delete the demo.txt file
rm -rf demo # delete the demo directory and all the files inside of it.  
rm -rf demo1 demo2 demo3 # delete more than one directory.

cp

cp demo.txt ~/desktop/ # copy the demo.txt file to the desktop 
cp demo.txt ~/desktop/demo1.txt # copy the demo.txt file to the desktop and rename it to demo1.txt
cp -r demo1 demo2 # copy the demo1 directory to the demo2 directory (no need to create demo2 beforehand)

-r will be needed if you want to copy a directory.


tar

tar -zcrf demo.tar ./demo.txt # compress the demo.txt and name the compressed file to demo.tar
tar -zcrf demo.tar ./demo # compress all the files inside of the demo directory and name the compressed file to demo.tar
tar -zxvf demo.tar # uncompress the demo.tar file to a directory called demo  
tar -zxvf demo.tar -C ./Aaron # uncompress the demo.tar file to a directory called Aaron (created beforehand)

find

find / -name demo.txt # find a file called demo.txt from the root directory
find ~/desktop -name demo.txt # narrow down the range of searching if you are sure that the demo.txt is inside of your desktop
find / -name demo* # find any file that starts with demo from the root directory
find / -name *.txt # find any file that ends with .txt from the root directory

If you are using a MacBook, you have to change the command to find / -name ‘demo.txt’ or find / -name “demo.txt”
